# Code Of Conduct

## Standards

Participants should be respectful, constructive, and focused on improving a defensive security project.

Good participation includes:

- using clear technical language
- assuming good faith
- accepting correction
- respecting different experience levels
- keeping discussion relevant to the project

Unacceptable behavior includes:

- harassment or threats
- personal attacks
- sharing private data without permission
- submitting or encouraging unauthorized access
- adding offensive, destructive, or credential-harvesting behavior

## Enforcement

Maintainers may edit, close, or remove issues, pull requests, comments, or contributions that violate this code of conduct or the project's defensive scope.
